cit-ում origin հեռավոր շտեմարանի ստեղծած անունն է,որին ի սկզբանե կլոնավորել է նախագիծը։Այն օգտագործվում է URL պահեստի  փոխարեն և դրանով ավելի է հեշտացնում հղումը։
Git merge-ը հրամանը միավորում է առանձին ճյուղեր և լուծում ցանկացած հակասական խմբագրում։
Երկու դեպպում միաձուլումը կարող է մտնել կոնֆլիկտային վիճակ։git conflict-ը մի իրավիճակ է որը տեղի է ունենում երբ  git-ը
չի կարողանում ավտեմատ կերպով լուծել երկու պարտավերությունների 
 միջև եղած հրամաներ
 git revert հրամանը օգտագործվուէ պահեստի պարտավորությունը պատմությունը չեղարկելու համար։<հետարկել>այլ հրամաներ ինչպիսիք են git reset,git checout, HEARD։
rebasing -ը նոր բազային հարցում տեղափոխելու կամ համակցելու գործնթաց է։rebasing-ը առավել է   օգտակար է և հեշտությամբ է պատկերացվում  գործառույթների ճյուղավորման և աշխատանքային հոսքի համատեքստում։Նրա հիմնական առավոլություն կայանում է նրանում որ դուք ստանում ենք ավելի մաքուր նախագիծ։Նախ այն վերացնում է  Git merge-ի կողմից պահնջվող անհարկի միաձուլման պարտավորությունները։
